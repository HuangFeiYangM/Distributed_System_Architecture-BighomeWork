## 3. 测试范围与顺序

### 3.1 测试模块划分

| 模块 | 测试内容 | 优先级 |
|------|---------|--------|
| 用户服务 | 注册、登录、Token刷新、信息查询/修改 | P0 |
| 菜品服务 | 菜品CRUD、菜单发布、今日菜单查询、库存扣减/恢复 | P0 |
| 订单服务 | 下单、订单查询、状态流转、超时取消、取餐回调 | P0 |
| 取餐服务 | 入队、叫号、核销、队列查询、大屏数据、WebSocket | P0 |
| 网关 | JWT校验、路由转发、白名单、WebSocket代理 | P1 |

### 3.2 推荐测试顺序（存在依赖关系）

```
Step 1: 用户服务（注册 → 登录获取Token）
    ↓
Step 2: 菜品服务（商家登录 → 创建菜品 → 发布菜单）
    ↓
Step 3: 订单服务（用户登录 → 浏览菜单 → 下单）
    ↓
Step 4: 取餐服务（商家接单 → 制作 → 完成 → 叫号 → 核销）
    ↓
Step 5: 边界与异常测试（超时、库存不足、无权限等）
```

---

### 4.2 TC-U02：用户登录

| 项 | 内容 |
|----|------|
| **接口** | `POST /api/user/login` |
| **描述** | 使用手机号+密码登录，获取 JWT Token |

**请求体**：
```json
{
  "phone": "13800000002",
  "password": "user123"
}
```

**预期结果**：
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIs...",
    "tokenType": "Bearer",
    "expiresIn": 86400,
    "userId": 3,
    "role": 0
  },
  "timestamp": 1714291200000
}
```

> **重要**：记录返回的 `accessToken`，后续所有需要鉴权的接口都要在 Header 中携带：
> ```
> Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
> ```

## 11. 使用 Python 脚本测试

项目提供了 `tests/run_backend_api_tests.py`，使用标准库（不依赖第三方库），会按“尽量跑通”的路径调用一组接口，并在每一步打印响应（关键 ID 提取失败时会提前停止）。

从仓库根目录执行（网关默认 `127.0.0.1:8080`）：
```bash
python tests\run_backend_api_tests.py --base-url http://127.0.0.1:8080
```

常用参数：
- `--skip-registration`：不自动注册，只尝试登录
- `--register-if-login-fail`：登录失败则注册一次后重试
- `--sleep 0.6`：每一步之间的等待秒数

## 13. 库存接口专项脚本（新增）

新增脚本：`tests/run_backend_api_stock_tests.py`

覆盖范围：
- 商家分页查询自己库存：`GET /api/menu/stock/merchant`
- 管理员分页查询全局库存：`GET /api/menu/stock/list`
- 商家修改自己库存（`INCR`）成功
- 普通用户修改库存被拒绝（403 语义）
- 非法大幅 `DECR` 触发冲突业务码（`2002` 或 `2004`）

执行命令：
```bash
python tests\run_backend_api_stock_tests.py --base-url http://127.0.0.1:8080
```

说明：
- 脚本会从 `GET /api/menu/today` 自动提取 `menuDishId`；若无今日菜单会直接失败并提示先发布菜单。
- 返回统一按 `code` 判断，库存冲突为业务失败（HTTP 可能仍为 200）。
